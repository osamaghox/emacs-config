* Introduction

هذا الملف يحتوي على إعداداتي الشخصية لـ Emacs باستخدام Org Mode،
حيث كل كود بلغة Emacs Lisp (elisp) يتم تصديره تلقائيًا إلى ملف init.el عند حفظ ملف الـ config.org

كما يتضمن شرحًا مفصلًا لإعداد نسخة محمولة من Emacs على نظام Windows،
بالإضافة إلى تكامل Syncthing لمزامنة الإعدادات بين جميع الأجهزة.

** Explanation of Export and Automatic Execution Settings

هذا القسم يشرح سطرين مهمين في أعلى الملف يقومان بتفعيل بعض السلوكيات المفيدة تلقائيًا عند فتح أو حفظ الملف.

*** Local Variables for the File

#+begin_example
# -*- after-save-hook: (org-babel-tangle); before-save-hook: (delete-trailing-whitespace)-*-
#+end_example

- هذا السطر يُخبر Emacs بأن ينفّذ بعض الأوامر عند حفظ الملف أو قبل حفظه.
- `after-save-hook: (org-babel-tangle)`
  ⇒ بعد حفظ الملف، يقوم Emacs تلقائيًا بتشغيل `org-babel-tangle`، أي يتم استخراج كود Emacs Lisp من الملف إلى ملف `.el`.
- `before-save-hook: (delete-trailing-whitespace)`
  ⇒ قبل الحفظ، يتم حذف الفراغات الزائدة في نهاية الأسطر تلقائيًا.

*** Header Arguments for Code Blocks

#+begin_example
#+PROPERTY: header-args:emacs-lisp :exports code :results none :tangle ~/.emacs.d/init.el :mkdirp yes
#+end_example

- هذا السطر يحدد الخصائص الافتراضية لجميع كتل الكود من نوع `emacs-lisp` في الملف.
- `:exports code` ⇒ عند تصدير الملف (مثل PDF أو HTML)، سيتم تصدير الكود فقط بدون النتائج.
- `:results none` ⇒ لا يتم عرض نتائج تنفيذ الكود داخل الملف.
- :tangle ~/.emacs.d/init.el ⇒ سيتم تصدير الكود إلى الملف init.el في مجلد .emacs.d
- `:mkdirp yes` ⇒ إذا لم يكن المجلد موجودًا، سيتم إنشاؤه تلقائيًا.

*** Result

عند حفظ الملف `config.org`:
1. يتم تنظيفه من الفراغات الزائدة.
2. يتم تحديث ملف `init.el` تلقائيًا بالكود المكتوب داخل كتل `#+begin_src emacs-lisp`.

*** Installing AutoIt

AutoIt هي أداة مجانية على نظام Windows تُستخدم لإنشاء سكربتات بسيطة لأتمتة المهام، مثل تشغيل البرامج، إنشاء اختصارات، التعامل مع الملفات، وتعيين متغيرات بيئة النظام.

سنستخدم AutoIt لإنشاء سكربت يقوم بما يلي:
- تشغيل Emacs من مسار محدد.
- تعيين متغير البيئة `HOME` إلى المجلد الحالي، لجعل Emacs يعمل كنسخة محمولة (Portable).
- إنشاء اختصار تلقائي في قائمة Start Menu باسم يحتوي على اسم المجلد الأب (لتمييز النسخة).

**** Creating the Script File

بعد تثبيت AutoIt، نقوم بإنشاء ملف جديد بالامتداد `.au3`، وليكن اسمه مثلًا:
EmacsPortableLauncher.au3

#+begin_example

#include <File.au3>

Global $CurrentDir = @ScriptDir
Global $ParentDir = StringLeft($CurrentDir, StringInStr($CurrentDir, "\", 0, -1) - 1)
Global $FolderName = StringTrimLeft($ParentDir, StringInStr($ParentDir, "\", 0, -1))

; اختصار السكربت نفسه
Global $ScriptPath = @ScriptFullPath
Global $ShortcutName = "Emacs (" & $FolderName & ").lnk"
Global $ShortcutPath = @ProgramsDir & "\" & $ShortcutName

; إنشاء أو تحديث الاختصار للسكربت
If FileExists($ShortcutPath) Then
    Local $sc = FileGetShortcut($ShortcutPath)
    If @error _
       Or StringLower($sc[0]) <> StringLower($ScriptPath) _
       Or StringLower($sc[1]) <> StringLower($CurrentDir) Then
        FileCreateShortcut($ScriptPath, $ShortcutPath, $CurrentDir, "", "Emacs Portable - " & $FolderName)
    EndIf
Else
    FileCreateShortcut($ScriptPath, $ShortcutPath, $CurrentDir, "", "Emacs Portable - " & $FolderName)
EndIf

; تعيين HOME وتشغيل Emacs
EnvSet("HOME", $CurrentDir)
Run('"' & $CurrentDir & '\runemacs.exe"', $CurrentDir)

#+end_example

ثم نحفظ ملف السكربت ووضعه بجوار
runemacs.exe الي يوجد في مجلد bin

** Android Port

*** Objective
تشغيل Emacs كواجهة رسومية على نظام أندرويد (دون الاعتماد على Termux فقط)، مع إمكانية استخدام بيئة GNU/Linux من خلال Termux، ومزامنة الإعدادات بين الهاتف والكمبيوتر باستخدام أداة SyncThing.

*** Removing Old Apps
- احذف أي نسخة سابقة من =Emacs=.
- احذف تطبيق =Termux= العادي إذا كنت قد ثبّته من F-Droid أو GitHub.

*** Installing Correct Versions in Order
1. ثبّت أولاً نسخة =Termux= المعدلة من:
   [[https://sourceforge.net/projects/android-ports-for-gnu-emacs/files/termux/][Termux SourceForge]]
2. افتح تطبيق =Termux= مرة واحدة (مهم جدًا).
3. ثبّت بعدها =Emacs الرسومي= من:
   [[https://sourceforge.net/projects/android-ports-for-gnu-emacs/files/][Emacs GUI SourceForge]]
4. بعد التثبيت، افتح تطبيق =Emacs=.

*** Granting Full Permissions
ادخل إلى إعدادات نظام أندرويد ثم:
- إعدادات > التطبيقات > Emacs وTermux > فعّل جميع الصلاحيات التالية:
  - صلاحية الوصول الكامل إلى جميع الملفات (لإصدارات Android 11 فأعلى): =Special app access > All files access=
  - صلاحية الوصول إلى التطبيقات المثبّتة.
  - صلاحيات الخلفية والإشعارات، وغيرها حسب الحاجة.

*** Clean Symbolic Links from The Home Directory Before Creating New Ones

قبل تنفيذ أوامر `mkdir` و `ln -s`، من الأفضل التأكد من أن مجلد `~` لا يحتوي على روابط رمزية غير ضرورية قد تسبب تعارضًا لاحقًا.

1. عرض جميع الروابط الرمزية فقط داخل مجلد Home (~):

#+BEGIN_EXAMPLE
find ~ -maxdepth 1 -type l
#+END_EXAMPLE

2. حذف جميع الروابط الرمزية داخل ~ فقط (وليس محتوى المجلدات الأصلية):

#+BEGIN_EXAMPLE
find ~ -maxdepth 1 -type l -exec rm -v {} \;
#+END_EXAMPLE

*** Linking Android Folders Inside Emacs

من داخل الطرفية في Emacs، نفّذ أوامر الربط الرمزي التالية:

#+BEGIN_EXAMPLE
# إنشاء مجلد الوجهة إذا لم يكن موجودًا
mkdir -p /data/data/org.gnu.emacs/files

# إنشاء مجلدات المصدر على sdcard إذا مش موجودة
mkdir -p /sdcard/.emacs.d
mkdir -p /sdcard/sync-thing
mkdir -p /sdcard/sync-thing/online-sync/emacs/emacs-data/fonts

# رابط sdcard/.emacs.d
rm -rf /data/data/org.gnu.emacs/files/.emacs.d
ln -s /sdcard/.emacs.d /data/data/org.gnu.emacs/files/.emacs.d

# رابط sdcard/sync-thing
rm -rf /data/data/org.gnu.emacs/files/sync-thing
ln -s /sdcard/sync-thing /data/data/org.gnu.emacs/files/sync-thing

# إنشاء مجلد fonts قبل الربط
mkdir -p /data/data/org.gnu.emacs/files/fonts

# رابط مجلد الخطوط
rm -rf /data/data/org.gnu.emacs/files/fonts
ln -s /sdcard/sync-thing/online-sync/emacs/emacs-data/fonts /data/data/org.gnu.emacs/files/fonts
#+END_EXAMPLE

* Syncthing Installation & Configuration Guide

هو برنامج *مزامنة ملفات مفتوح المصدر* يعمل بطريقة *نظير إلى نظير (P2P)*، مما يسمح لك بمزامنة الملفات والمجلدات بين أجهزتك المختلفة *مباشرة دون الحاجة إلى خوادم سحابية* مثل Google Drive أو Dropbox.

** How Syncthing Works

سنستخدم برنامج *Syncthing* لمزامنة ملفات Emacs بين جهازين حاسوب وهاتف محمول، بطريقة لا تتطلب اتصالًا مباشرًا بين الجهازين.

*** Phone as a "Central Hub"

لن يتم ربط الجهازين مباشرة، بل سيتم الربط على النحو التالي:

  - *PC1 (حاسوب العمل)* → *الهاتف*
  - *PC2 (حاسوب المنزل)* → *الهاتف*

ولا يوجد اتصال مباشر بين:

  - *PC1* ↔ *PC2*

هذه الطريقة تفيد في الحالات التالية:

  - عند عدم توفر اتصال دائم بين الحاسوبين.
  - أو عند استخدام شبكات مختلفة (مثلاً العمل والبيت).
  - أو إذا كنت تريد استخدام الهاتف كنقطة تحكم ومزامنة أساسية.

*** General Idea

  1. الهاتف يكون دائمًا قيد التشغيل ومتصلاً بالإنترنت.
  2. عند تشغيل أي من الحواسيب، يتم المزامنة مع الهاتف فقط.
  3. الهاتف بدوره يقوم بنقل التغييرات إلى الحاسوب الآخر عند اتصاله لاحقًا.

** Windows

تأكد من تطبيق هذه الخطوات على جميع أجهزة Windows التي ترغب في مزامنتها لضمان عمل Syncthing بشكل صحيح وفعّال.

1. نقوم بتنزيل البرنامج للحاسوب من الرابط التالي
   https://github.com/syncthing/syncthing/releases
   syncthing-windows-amd64-x.x.x-rc.1.zip

*** Creating Two Folders for Syncthing

1. نقوم بإنشاء مجلدين باسمين مختلفين **ويجب أن تكون الأسماء كما هي بالضبط**، ثم نضع ملفات البرنامج داخل كل مجلد على حدة.
الأسماء الضرورية للمجلدين هي:

- *SyncthingPortableLan*

هذه النسخة تقتصر على المزامنة داخل الشبكة المحلية فقط (LAN)، ولا تستخدم الإنترنت.

- *SyncthingPortable*

هذه نسخة تقوم بالمزامنة عبر الإنترنت وأيضًا عبر نفس الشبكة المحلية.
مزامنة الملفات الكبيرة عبر الإنترنت قد تستهلك كمية كبيرة من البيانات، فانتبه لذلك.

هذا الإجراء مهم حتى لا يتكرر اختصار التشغيل عند بدء النظام ولا يتكرر الاختصار في قائمة التشغيل.

2. نسخ الملف [[file:Syncthing Portable Launcher.exe]] إلى كلا المجلدين.

*** AutoIt Script (For Backup Only)

#+begin_src autoit
#include <File.au3>
#include <MsgBoxConstants.au3>
#include <GUIConstantsEx.au3>
#include <WindowsConstants.au3>
#include <Array.au3>

; —————— وسيط جديد لفتح المتصفح يدوياً ——————
Global $openBrowser = False
If $CmdLine[0] > 0 And StringLower($CmdLine[1]) = "-open" Then
    $openBrowser = True
EndIf

; —————— المتغيرات الأساسية ——————
Global $launcherPath     = @ScriptFullPath
Global $currentDir       = @ScriptDir
Global $folderName       = StringTrimLeft($currentDir, StringInStr($currentDir, "\", 0, -1))
Global $startupPath      = @StartupDir
Global $shortcutName     = "Syncthing Portable Launcher_" & $folderName & ".lnk"
Global $shortcutFullPath = $startupPath & "\" & $shortcutName
Global $syncthingExe     = $currentDir & "\syncthing.exe"
Global $userDataDir      = $currentDir & "\user data"
Global $configFile       = $userDataDir & "\config.xml"
Global $firstRunFlag     = $userDataDir & "\firstrun.flag"
; مسار قائمة Start Menu للمستخدم الحالي
Global $programsPath     = @ProgramsDir
Global $smShortcutName   = $folderName & ".lnk"
Global $smShortcutPath   = $programsPath & "\" & $smShortcutName

; —————— أنشئ مجلّد البيانات إذا لم يكن موجوداً ——————
If Not FileExists($userDataDir) Then DirCreate($userDataDir)

; —————— إنشاء/تحديث الاختصار في Startup بدون وسائط ——————
If FileExists($shortcutFullPath) Then
    Local $sc = FileGetShortcut($shortcutFullPath)
    If @error _
       Or StringLower($sc[0]) <> StringLower($launcherPath) _
       Or StringLower($sc[1]) <> StringLower($currentDir) Then
        FileCreateShortcut($launcherPath, $shortcutFullPath, $currentDir)
    EndIf
Else
    FileCreateShortcut($launcherPath, $shortcutFullPath, $currentDir)
EndIf

; —————— إنشاء/تحديث الاختصار في قائمة Start Menu مع الوسيطة -open ——————
If FileExists($smShortcutPath) Then
    Local $sm = FileGetShortcut($smShortcutPath)
    ; $sm[0]=target, [1]=working dir, [2]=args
    If @error _
       Or StringLower($sm[0]) <> StringLower($launcherPath) _
       Or StringLower($sm[1]) <> StringLower($currentDir) _
       Or StringLower($sm[2]) <> "-open" Then
        FileCreateShortcut($launcherPath, $smShortcutPath, $currentDir, "-open", "", "", $folderName)
    EndIf
Else
    FileCreateShortcut($launcherPath, $smShortcutPath, $currentDir, "-open", "", "", $folderName)
EndIf

; —————— دوال مساعدة ——————
Func PortInUse($port)
    Return RunWait(@ComSpec & " /c netstat -aon | findstr :" & $port, "", @SW_HIDE) = 0
EndFunc

Func _ProcessCloseByName($procName)
    Local $pList = ProcessList($procName)
    For $i = 1 To $pList[0][0]
        ProcessClose($pList[$i][1])
    Next
    Sleep(1000)
    $pList = ProcessList($procName)
    For $i = 1 To $pList[0][0]
        ProcessKill($pList[$i][1])
    Next
EndFunc

Func GetGuiPortFromConfig()
    If Not FileExists($configFile) Then Return 0
    Local $lines = FileReadToArray($configFile)
    If @error Then Return 0
    For $i = 0 To UBound($lines)-1
        If StringRegExp($lines[$i], '<address>127\.0\.0\.1:(\d+)</address>') Then
            Local $m = StringRegExp($lines[$i], '<address>127\.0\.0\.1:(\d+)</address>', 1)
            If IsArray($m) Then Return (Number($m[0]) = 8384) ? 0 : Number($m[0])
        EndIf
    Next
    Return 0
EndFunc

Func FixConfigPorts()
    Local $guiPort    = 0, $listenPort = "", $lines = [], $changed = False
    Local $isFirstRun = Not FileExists($firstRunFlag)

    If FileExists($configFile) Then
        $lines = FileReadToArray($configFile)
        If @error Then $lines = []
    EndIf

    If $isFirstRun Then
        Local $res = ShowPortInputGUI()
        $guiPort    = $res[0]
        $listenPort = $res[1]
        Local $hFlag = FileOpen($firstRunFlag, 2)
        FileWrite($hFlag, "initialized")
        FileClose($hFlag)
    Else
        $guiPort = GetGuiPortFromConfig()
        If $guiPort = 0 Then $guiPort = 8385
        For $i = 0 To UBound($lines)-1
            If StringRegExp($lines[$i], '<listenAddress>tcp://.*:(\d+)</listenAddress>') Then
                Local $m = StringRegExp($lines[$i], '<listenAddress>tcp://.*:(\d+)</listenAddress>', 1)
                If IsArray($m) Then
                    $listenPort = $m[0]
                    ExitLoop
                EndIf
            EndIf
        Next
    EndIf

    If UBound($lines) = 0 Then
        Local $cfg = '<?xml version="1.0" encoding="utf-8"?>' & @CRLF & _
                    '<configuration>' & @CRLF & _
                    '  <gui enabled="true" tls="false">' & @CRLF & _
                    '    <address>127.0.0.1:' & $guiPort & '</address>' & @CRLF & _
                    '    <browser>never</browser>' & @CRLF & _
                    '  </gui>' & @CRLF & _
                    '  <options>' & @CRLF & _
                    '    <listenAddress>' & ($listenPort = "" ? "default" : "tcp://0.0.0.0:" & $listenPort) & '</listenAddress>' & @CRLF & _
                    '  </options>' & @CRLF & _
                    '</configuration>'
        Local $f = FileOpen($configFile, 2)
        FileWrite($f, $cfg)
        FileClose($f)
        Return
    EndIf

    For $i = 0 To UBound($lines)-1
        If StringRegExp($lines[$i], '<address>127\.0\.0\.1:\d+</address>') Then
            $lines[$i] = StringRegExpReplace($lines[$i], '<address>127\.0\.0\.1:\d+</address>', '<address>127.0.0.1:' & $guiPort & '</address>')
            $changed = True
        EndIf
        If StringRegExp($lines[$i], '<browser>.*</browser>') Then
            $lines[$i] = '    <browser>never</browser>'
            $changed = True
        EndIf
        If StringRegExp($lines[$i], '<listenAddress>.*</listenAddress>') Then
            $lines[$i] = '    <listenAddress>' & ($listenPort = "" ? "default" : "tcp://0.0.0.0:" & $listenPort) & '</listenAddress>'
            $changed = True
        EndIf
    Next

    Local $idxGui = _ArraySearch($lines, '<gui.*>', 0, 0, 2)
    If $idxGui <> -1 And Not StringRegExp($lines[$idxGui+1], '<browser>') Then
        _ArrayInsert($lines, $idxGui + 1, '    <browser>never</browser>')
        $changed = True
    EndIf

    Local $idxOpt = _ArraySearch($lines, '<options>', 0, 0, 2)
    If $idxOpt <> -1 And Not StringRegExp($lines[$idxOpt+1], '<listenAddress>') Then
        _ArrayInsert($lines, $idxOpt + 1, '    <listenAddress>' & ($listenPort = "" ? "default" : "tcp://0.0.0.0:" & $listenPort) & '</listenAddress>')
        $changed = True
    EndIf

    If $changed Then
        Local $f = FileOpen($configFile, 2)
        For $i = 0 To UBound($lines)-1
            FileWriteLine($f, $lines[$i])
        Next
        FileClose($f)
    EndIf
EndFunc

  هذا هو المنفذ الذي يستخدمه Syncthing للتواصل مع الأجهزة الأخرى.
  - SyncthingPortable: 28000 (كمثال)
  - SyncthingPortableLan: 29000 (كمثال)

** Important Notes About Syncthing

- **لا تحذف ملف `.stignore`** من مجلد `.emacs.d`، لأنه يحتوي على إعدادات حيوية لتجاهل ملفات معينة.
- Syncthing يعمل بالمزامنة ثنائية الاتجاه، لذا أي تغيير في جهاز ينعكس على الآخر.
- إذا كان مجلد `sync-thing` فارغًا على الحاسوب ومليئًا على الهاتف، قد يتم حذف الملفات من الهاتف.
- عند تشغيل ملف من الهاتف على الحاسوب، قد تظهر رسالة Out of Sync لأن الملف قيد الاستخدام. يجب إغلاق البرنامج المشغّل لإتمام المزامنة.
- حافظ على اتصال الأجهزة أثناء التعديل كلما أمكن.
- استخدم ميزة "نسخ الإصدار" (Versioning) لحفظ النسخ القديمة وعدم حذفها.
- تجنب تعديل نفس الملفات على أجهزة غير متصلة لفترات طويلة أو في آن واحد.
- عند الاتصال بنفس الشبكة، لا يتم استخدام الإنترنت.
- الهاتف هو الجهاز الرئيسي لحفظ البيانات، تأكد من إتمام المزامنة عبره.
- قم بعمل نسخة احتياطية شهرية على الكلاود أو فلاشة، بما في ذلك المجلد المشترك على الهاتف.

* Very Important: Configure init.el with lexical-binding

في هذا الجزء نهيّئ ملف إعدادات Emacs الرئيسي (`init.el`) مع تفعيل *الربط المعجمي* (`lexical-binding`) لتحسين الأداء والتحكم الأفضل في المتغيرات.

كذلك، نمنع ظهور رسالة الترحيب عند تشغيل Emacs لجعل الواجهة أنظف وأسرع.

(setq debug-on-error t)           ;; تفعيل المصحح عند حدوث خطأ
#+end_src

*** What Does This Message Mean?

هذه الرسالة تعني أن الملف يحتوي على متغيرات محلية (Local Variables) قد تُعتبر "غير آمنة" لأن بعضها ينفذ أوامر تلقائيًا، مثل:

- `org-babel-tangle` (لتصدير الأكواد)
- `delete-trailing-whitespace` (لحذف الفراغات الزائدة)

Emacs يعرض هذا التحذير ويسألك إن كنت ترغب بالسماح بتنفيذ هذه الأوامر (`y`) أو تجاهلها (`n`).

*** How to Automatically Bypass This Message?

يمكننا تجاوز هذه الرسالة وعدم ظهورها مرة أخرى عن طريق إضافة الكود التالي داخل ملف الإعدادات `init.el` أو أي ملف Emacs Lisp يتم تحميله مع Emacs:

#+begin_src emacs-lisp
(setq safe-local-variable-values
      '((after-save-hook . (org-babel-tangle))
        (before-save-hook . (delete-trailing-whitespace))))
#+end_src

هذا الكود يقوم بإخبار Emacs أن هذه القيم المحلية (Local Variables) آمنة، وبالتالي لن يسألك مرة أخرى عند فتح ملفات تحتوي عليها.

** How to Add Code to init.el Using Org-mode?

لإضافة كود إلى ملف `init.el` من داخل ملف `config.org`، نقوم بكتابة **كتلة كود (src block)** من نوع `emacs-lisp`، مثل المثال التالي:

#+begin_src emacs-lisp
;; هنا نكتب الكود الذي نريده
#+end_src

أي كود نضعه داخل هذه الكتلة سيتم تصديره (tangle) تلقائيًا إلى ملف `init.el` عند الحفظ من خلال `C-x C-s`، إذا كنا قد فعلنا خاصية التانغل التلقائي.
تسمى هذه "كتلة كود" أو بالإنجليزية: **code block** أو **src block**، وليست "كود" بحد ذاتها.

*** Better Font Faces

الدالة =osama/org-font-setup= تقوم بضبط مظهر الخطوط في Org Mode، بحيث:

(defun osama/org-font-setup ()

                             (0 (prog1 () (compose-region (match-beginning 1) (match-end 1) "•"))))))

                    (org-level-2 . 1.1)
                    (org-level-3 . 1.05)
                    (org-level-4 . 1.0)
                    (org-level-5 . 1.1)
                    (org-level-6 . 1.1)
                    (org-level-7 . 1.1)
                    (org-level-8 . 1.1)))
      (set-face-attribute (car face) nil
                          :font osama/preferred-font
                          :weight 'bold
                          :height (cdr face)))))

*** Toggle RTL/LTR/Auto Direction

في هذا الجزء، سننشئ دالة في Emacs Lisp لتبديل اتجاه الفقرة في org-mode بين:

1. اليسار إلى اليمين (LTR)
2. اليمين إلى اليسار (RTL)
3. الوضع التلقائي (Auto): يحدد الاتجاه حسب أول حرف من الفقرة.
#+begin_src emacs-lisp
;; تفعيل إعادة ترتيب النصوص للغات التي تكتب من اليمين لليسار
(setq-default bidi-display-reordering t)

;; الوضع الافتراضي لكل الملفات (اتجاه الفقرة تلقائي)
(setq-default bidi-paragraph-direction nil)

(defun osama/toggle-bidi-paragraph-direction ()
  "Toggle paragraph direction between LTR, RTL, and Auto in any Emacs mode."
  (interactive)
  (setq bidi-paragraph-direction
        (cond
         ((eq bidi-paragraph-direction nil) 'left-to-right)
         ((eq bidi-paragraph-direction 'left-to-right) 'right-to-left)
         ((eq bidi-paragraph-direction 'right-to-left) nil)))
  (message "Paragraph direction now: %s"
           (pcase bidi-paragraph-direction
             ('left-to-right "LTR")
             ('right-to-left "RTL")
             (_ "Auto")))

  (recenter))

;; تعيين مفتاح اختصار عام لتبديل الاتجاه في كل أوضاع Emacs

* System Selector Settings

هذا القسم يتيح لك تحميل إعدادات خاصة حسب النظام الذي تستخدمه (مثل: Android، Linux، Windows...).
الفكرة أن يكون لديك ملف إعدادات مستقل لكل نظام داخل مجلد `~/.emacs.d/`، يبدأ اسمه بـ `init-`، ويتم اختياره تلقائيًا عند تشغيل Emacs.

#+begin_src emacs-lisp
(let* ((init-dir "~/.emacs.d/")
       (init-files (directory-files init-dir nil "^init-.*\\.el$"))
       (options (mapcar (lambda (f)
                          (string-remove-suffix ".el" (string-remove-prefix "init-" f)))
                        init-files))
       (choice (completing-read "أي إعداد تريد تحميله؟ " options nil t))
       (settings-file (expand-file-name (concat "init-" choice ".el") init-dir)))
  (when (file-exists-p settings-file)
    (load-file settings-file)))
#+end_src

** Important Notes

- لكل نظام (مثل "Linux" أو "Android") يجب أن يكون عنوان من المستوى الأول `*`.
- مباشرة **بعد العنوان**، يجب وضع خصائص التانجل داخل `:PROPERTIES:` دون فواصل أو أسطر فارغة.
- كل كود داخل القسم يجب أن يكون تحت عنوان فرعي `**`.

لإضافة إعدادات لنظام جديد، اتبع الخطوات التالية:

1. أضف عنوانًا من المستوى الأول باسم النظام، مثل:

   `* إعدادات Linux`

2. مباشرة بعد العنوان، أضف خصائص التانجل:

** Media Handling in org-mode

- نسخ الصور (مثل PNG وJPG) إلى assets/images/
- نسخ الفيديوهات (مثل MP4 وMOV) إلى assets/videos/
- نسخ الملفات الأخرى (مثل ZIP وPSD وPDF...) إلى assets/random/
- إدراج رابط مباشر للملف داخل ملاحظتي.
- هذا يسهل تنظيم المرفقات داخل المشروع بدل تراكمها في نفس مجلد الملاحظات.
- الصور والفيديوهات والملفات تُعاد تسميتها بناءً على الوقت الحالي (تاريخ + ساعة) وتُخزَّن تلقائيًا في مجلدها المخصص.
- ملاحظة: يتم إنشاء المجلدات تلقائيًا إذا لم تكن موجودة، ولا حاجة لتثبيت أدوات خارجية على Windows.

#+begin_src emacs-lisp
(use-package org-download
  :after org
  :config
  (defun osama/org-download-insert-function (link)
    "Custom handler for inserting downloads into appropriate folders under ~/sync-thing/online-sync/emacs/osama-org/assets."
    (let* ((ext (downcase (file-name-extension link)))
           (time (format-time-string "%Y%m%d-%H%M%S"))
           (filename (format "%s.%s" time ext))
           (base-dir (expand-file-name "~/sync-thing/online-sync/emacs/osama-org/assets"))
           (target-dir
            (cond
             ((member ext '("png" "jpg" "jpeg" "gif" "svg"))
              (expand-file-name "images" base-dir))
             ((member ext '("mp4" "mov" "avi" "mkv"))
              (expand-file-name "videos" base-dir))
             (t (expand-file-name "random" base-dir))))
           (target-path (expand-file-name filename target-dir)))
      (unless (file-exists-p target-dir)
        (make-directory target-dir t))
      (copy-file link target-path t)
      ;; ترجع اسم الملف النهائي ليتم إدراجه تلقائيًا في org-download
      target-path))

  (setq org-download-method 'osama/org-download-insert-function
        org-download-heading-lvl nil
        org-download-screenshot-method
        "powershell -command \"Add-Type -AssemblyName System.Windows.Forms; Add-Type -AssemblyName System.Drawing; $bmp = New-Object System.Drawing.Bitmap([System.Windows.Forms.Screen]::PrimaryScreen.Bounds.Width, [System.Windows.Forms.Screen]::PrimaryScreen.Bounds.Height); $graphics = [System.Drawing.Graphics]::FromImage($bmp); $graphics.CopyFromScreen(0, 0, 0, 0, $bmp.Size); $file = 'screenshot.png'; $bmp.Save($file, [System.Drawing.Imaging.ImageFormat]::Png); Write-Output $file\"")
